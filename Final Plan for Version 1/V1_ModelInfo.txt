## 1.1 AuthUser (Domain Entity)
Represents the authenticated user in the app.
*   **Path**: `lib/features/auth/domain/entities/auth_user.dart`
*   **Fields**:
    *   `String id` (Unique User ID from Firebase)
    *   `String? email`
    *   `String? phoneNumber`
    *   `String? displayName`
    *   `String? photoUrl`
    *   `bool emailVerified`
    *   `DateTime? lastSignIn`
    *   `String? provider` (google, phone, email, apple)
    *   `bool isAnonymous`
    *   `List<String>? customClaims`
*   **Methods**:
    *   `copyWith()`
    *   `toMap()` / `fromMap()` (for simple serialization if needed)
    *   `isFullyVerified()` (checks emailVerified + phone if applicable)

## 1.2 UserProfile (Database Entity)
Extended user data stored in Cloud Firestore (Users Collection).
*   **Path**: `lib/features/settings/domain/entities/user_profile.dart`
*   **Fields**:

    *   `String? kycTier` (Enum: BASIC, PRO, ENTERPRISE)
    *   `String? timeZone`
    *   `String? defaultLanguage` (en, bn)
    *   `DateTime? lastActiveAt`
    *   `String? deviceId`
    *   `DateTime createdAt`
    *   `DateTime updatedAt`
*   **Methods**:
    *   `fromJson(Map<String, dynamic> json)`
    *   `toJson()`
    *   `copyWith(...)`
    *   `updatePreferences(Map<String, dynamic> newPrefs)`
    *   `isActive()` (checks lastActiveAt within 30 days)

## 2.1 Contact (Local & Remote)
The core logic unit. Represents a saved contact.
*   **Path**: `lib/features/contacts/domain/entities/contact.dart`
*   **Fields**:
    *   `String id` (UUID)
    *   `String ownerId` (User who saved this contact)
    *   `String fullName` (Required)
    *   `String? jobTitle`
    *   `String? companyName`
    *   `List<String> phoneNumbers`
    *   `List<String> emails`
    *   `List<String> addresses`
    *   `List<String> websiteUrls`
    *   `List<SocialLink> socialLinks` (Custom Type)
    *   `String category` (Enum: Business, Personal, Friends, Family, Uncategorized)
    *   `List<String> tags`
    *   `bool isFavorite`
    *   `String? notes`
    *   `String? frontImageUrl` (Cloudinary URL - Front of card)
    *   `String? backImageUrl` (Cloudinary URL - Back of card)
    *   `String? frontImageOcrText` (Raw OCR text from front)
    *   `String? backImageOcrText` (Raw OCR text from back)
    *   `Map<String, dynamic>? ocrFields` (Structured OCR data)
    *   `DateTime? lastContactedAt`
    *   `int? contactCount` (How many times contacted)
    *   `String? source` (Enum: MANUAL, SCAN, IMPORT, NFC, QR)
    *   `bool isVerified` (Global directory verification status)
    *   `double? fraudScore` (0.0 to 1.0)
    *   `DateTime createdAt`
    *   `DateTime updatedAt`
    *   `String? createdBy` (User ID who created this contact)
    *   `List<String>? sharedWith` (List of user IDs for team sharing)
*   **Methods**:
    *   `fromJson(Map<String, dynamic> json)`
    *   `toJson()`
    *   `copyWith(...)`
    *   `matchesQuery(String query)` (Helper for search)
    *   `addPhoneNumber(String number)`
    *   `addEmail(String email)`
    *   `updateLastContacted()`
    *   `isBusinessContact()` (category == Business)

    *   `DateTime versionTimestamp`
    *   `bool verifiedByOwner`
    *   `double fraudScore`
    *   `int updateCount`
    *   `String? lastUpdaterId` (User ID who last updated)

*   **Methods**:

REQUIRED GENERATORS

The following packages will be used to generate boilerplate:
*   `freezed`: For immutable data classes and `copyWith`.

*   `riverpod_generator`: For providers.

*Note: Domain Entities should ideally be pure Dart. Data Transfer Objects (DTOs) in the Data Layer will extend or map to these entities and include the specific Isar/Firestore annotations.*

ADDITIONAL CONSIDERATIONS
================================================================================

## Implementation Priority Order:
1. **AuthUser** and **UserProfile** (Authentication & User Management)
2. **Contact** and **SocialLink** (Core Contact Management)
3. **Wallet** and **WalletTransaction** (Internal Ledger System)
4. **CardDesign** (Digital Business Card Creator)
5. **Team** and **TeamMember** (Basic Collaboration)
6. **GlobalContact** (Smart Intelligence & Sync)
7. **Notification** and **AppSettings** (User Experience)
8. **AppError** and **PaginatedResult** (Infrastructure)

## Database Strategy:
*   **Local (Isar)**: All entities for offline-first experience
*   **Remote (Firestore)**: UserProfile, Contact, Wallet, Team, GlobalContact
*   **Cloudinary**: Card images, avatars, media assets

## Security Considerations:
*   Sensitive fields (PIN, customClaims) should be encrypted
*   Use field-level security rules in Firestore
*   Implement data validation at domain layer
*   Add audit trails for critical operations

## Performance Optimizations:
*   Use lazy loading for large lists (Pagination)
*   Cache frequently accessed entities (UserProfile, AppSettings)
*   Index critical fields in Isar (userId, createdAt, category)
*   Implement selective field fetching in repositories

## Future Extensibility:
*   Consider abstract base classes for common fields (id, createdAt, updatedAt)
*   Plan for multi-currency support in Wallet entities
*   Design for potential integration with external contact sources
*   Keep extension points for advanced team features (roles, permissions)
