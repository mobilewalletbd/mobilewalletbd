2.1 ISAR SCHEMAS (lib/core/database/isar_schemas.dart)
-------------------------------------------------------
Total Schemas: 11 collections

Core Entities:
1. AuthUserIsar - Authenticated user data
   - Fields: uid, email, phoneNumber, displayName, photoUrl, emailVerified
   - Indexes: uid (unique)
   
2. UserProfileIsar - Extended user profile
   - Fields: uid, fullName, jobTitle, companyName, bio, avatarUrl, preferences
   - Enums: AccountStatus (active, suspended, pendingVerification)
   - Enums: KycTier (basic, pro, enterprise)
   - Indexes: uid (unique)

Contact Management:
3. ContactIsar - Business contact entity
   - Fields: contactId, ownerId, fullName, phoneNumbers, emails, addresses
   - Additional: socialLinks, tags, category, OCR data, card images
   - Enums: ContactCategory (business, personal, friends, family, uncategorized)
   - Enums: ContactSource (manual, scan, import, nfc, qr)
   - Indexes: contactId (unique), ownerId
   
4. GlobalContactIsar - Shared directory contact
   - Fields: globalId, latestData, version, verifiedByOwner, fraudScore
   - Indexes: globalId (unique)

Digital Card:
5. CardDesignIsar - Personal digital business card
   - Fields: cardId, userId, themeColor, layoutStyle, visibleFields
   - Enums: LayoutStyle (classic, modern, minimal)
   - Enums: QrCodeStyle (static, dynamic, custom)
   - Enums: CardStatus (active, draft, archived)
   - Indexes: cardId (unique), userId

Wallet & Transactions:
6. WalletIsar - User's internal balance
   - Fields: walletId, userId, balance, currency, status
   - Enums: WalletStatus (active, frozen)
   - Indexes: walletId (unique), userId
   
7. WalletTransactionIsar - Transaction ledger
   - Fields: transactionId, walletId, type, amount, status
   - Enums: TransactionType (credit, debit)
   - Enums: TransactionStatus (pending, completed, failed)
   - Enums: TransactionCategory (internalTransfer, manualAdd, reward)
   - Indexes: transactionId (unique), walletId, timestamp

Team Collaboration:
8. TeamIsar - Team entity
   - Fields: teamId, name, ownerId, members, description
   - Enums: TeamStatus (active, inactive, archived)
   - Indexes: teamId (unique), ownerId

Notifications:
9. NotificationIsar - User notifications
   - Fields: notificationId, userId, type, title, message
   - Enums: NotificationType (contactUpdate, payment, teamInvite, system)
   - Indexes: notificationId (unique), userId, createdAt

Settings:
10. AppSettingsIsar - User preferences
    - Fields: userId, theme, language, notificationsEnabled
    - Enums: ThemeMode (light, dark, system)
    - Enums: AppLanguage (en, bn)
    - Enums: ViewMode (grid, list)
    - Enums: CurrencyDisplay (symbol, code, name)
    - Indexes: userId (unique)

Sync Infrastructure:
11. SyncQueueIsar - Background sync queue
    - Fields: operation, entityType, entityId, entityData, status
    - Enums: SyncOperation (create, update, delete)
    - Enums: EntityType (userProfile, contact, cardDesign, wallet, etc.)
    - Enums: SyncStatus (pending, inProgress, completed, failed)
    - Indexes: createdAt

Common Features Across All Schemas:
- Sync tracking: needsSync, lastSyncedAt fields
- Timestamps: createdAt, updatedAt
- Type-safe enums for status values
- Proper indexing for query performance

2.3 FIRESTORE SECURITY RULES (firestore.rules)
----------------------------------------------
Security Features:
✓ Authentication required for all operations
✓ Owner-based access control
✓ Team member verification
✓ Field-level validation
✓ Read-only fields protection
✓ Global contact access restrictions

Rule Categories:
- User Profile Rules: Users can only access their own profiles
- Contact Rules: Owners and team members can access shared contacts
- Global Contact Rules: Read access for all, verified update only
- Card Design Rules: Owner-only access
- Wallet Rules: Owner-only, balance modification restricted
- Transaction Rules: Wallet owner access, no deletion allowed
- Team Rules: Member-based access, admin update rights
- Notification Rules: User-specific access
- App Settings Rules: Owner-only access
